# 네이버 AI 검색 대응 실행 가이드

> 한 키워드를 찾아서 발행까지 거치는 6단계.
> 각 단계의 WHY를 먼저 납득하고 행동에 들어간다.

---

## TL;DR

최근 2026년 5월 기준 네이버 AI는 키워드와 글을 기계적으로 매칭하지 않는다. 

**검색자가 처한 상황과 목적**을 파악해서 가장 잘 맞는 글을 추천한다. 

그러므로 작업 순서도 역순으로 시작한다. 

글을 먼저 쓰는 게 아니라, 누가 읽을지 먼저 정한다.

블로그 지수, 글자 수, 키워드 반복 횟수는 이 게임의 점수판이 아니다. 
이제는 다른 게임의 룰이다.

---

## 빠른 의사결정 흐름 {#flow}

```
키워드 수령
    │
    ▼
[STEP 1] 키워드 진단 ─── 영역 판정 ─┬─ 1번 (스마트블록+플레이스) → 기존 전략 적용
    │                               ├─ 3번 (인기글·블탭) → 기존 블탭 전략 적용
    │                               ├─ 4번 (AI 브리핑) → 명확한 답변 구조 집중
    │                               └─ 2번 (새로운 스마트블록) → 아래 계속 ↓
    │
    ▼
[STEP 2] 경쟁 분석 ─── 상위 5개 플랫폼 분포 파악
    │                   │
    │                   ├─ 블로그 0개 → 강력추천영역 진입 포기 → 6등 이하 설계로 전환
    │                   └─ 블로그 있음 → 아래 계속 ↓
    │
    ▼
[STEP 3] 페르소나 설계 ─── 타겟 1그룹 확정 → 5W1H + Pain Point 도출
    │
    ▼
[STEP 4] 콘텐츠 설계 ─── 제목 패턴 결정 + FAQ/Q&A 구조 결정 + 권위성 설계
    │
    ▼
[STEP 5] 작성·교정 ─── 7단계 AI 워크플로 실행 + 사람 교정 필수
    │
    ▼
[STEP 6] 발행·운영 ─── 외부링크 안전성 점검 + 노출 모니터링
```

## STEP 1. 키워드 진단 {#step-1}

### 💡 WHY

영역을 잘못 판정하면 전략이 처음부터 틀린다. 새로운 스마트블록용 전략을 블탭 키워드에 쓰거나, 블로그 진입이 구조적으로 불가능한 키워드에 시간을 쏟는 실수가 여기서 발생한다. 30초 확인으로 수 시간의 낭비를 막는다.

### 🎯 ACTION

**1. 키워드를 네이버에서 직접 검색한다.**

**2. 화면 패턴으로 영역을 판정한다.**

| 번호 | 화면 패턴 | 영역 판정 | 전략 방향 |
|:----:|-----------|----------|----------|
| **1** | 지도/플레이스 블록 포함 | **1번 — 스마트블록 + 플레이스** | 기존 스마트블록 전략 |
| **2** | 5~8개 결과 묶음 + "더보기" 없음 | **2번 — 새로운 스마트블록 (강력추천영역)** | 이 자료의 주 대상 |
| **3** | "더보기" 없이 페이지 번호 (1/2/3/4/5...) | **3번 — 블탭+지식인 혼재 (인기글)** | 기존 블탭 전략 |
| **4** | 상단 AI 요약 박스 | **4번 — AI 브리핑** | 명확한 1:1 답변 구조 집중 |
| **5** | 쇼핑 광고 + 정보글 + 네이버스토어 상품 혼재, 하단에 추가 스마트블록 | **5번 — 쇼핑 광고 + 스마트블록 혼합형** | 정보글 + 구매 연결 동시 설계. 상품리뷰·쇼핑 연계 키워드에서 자주 발생 |
| ※ | 5~8개 결과 묶음 + "더보기" 버튼 있음 + 닉네임 자리에 `blog.naver.com/아이디` | **웹문서 노출** (별도 처리) | 블로그 노출과 구분해 분석. 어느 영역과도 별개로 식별 |

**3. 키워드 명명 형태로 영역을 미리 예측한다. (30초 선행 휴리스틱)**

키워드를 받는 순간, 검색 전에 형태만 보고 영역을 먼저 예측할 수 있다.

| 명명 형태 | 예시 | 예측 영역 |
|----------|------|----------|
| **이름형** (브랜드명·병원명 등) | ○○치과, ○○의원 | 통합검색 (브랜드 검색) |
| **행위형** (시술명·서비스명 등) | ○○ 보톡스, ○○ 시술 | 인기글 (정보 탐색형) |

예측 후 실제 검색 결과와 대조해 확인한다. 예측과 실제가 다르면 그 이유를 분석한다.

**4. 결과 수(5개인지 6개인지)로 판단하지 않는다.** 결과 수는 최소 5개~최대 8개로 가변이며, 조합 유형만 17종이다. 판단 기준은 더보기 유무와 표시 방식이다.

### 🔍 CHECK

- 영역 번호(1/2/3/4/5)가 확정됐는가?
- 웹문서 노출과 블로그 노출을 혼동하지 않았는가? (닉네임 vs 도메인 주소로 구분)
- 2번 영역으로 확정됐다면 STEP 2로 진행한다.

### ⚠ WATCH OUT

- 블탭 탭을 클릭해서 본 결과를 통합검색 결과로 착각하기 쉽다. 반드시 **통합검색 화면** 기준으로 판정한다.
- 웹문서로 노출된 블로그 글은 블로그 노출이 아니다. 닉네임 대신 `blog.naver.com/아이디`가 회색으로 보이면 웹문서다.
- "더보기"가 없어 강력추천영역처럼 보여도, 플레이스 블록이 섞여 있으면 1번 영역이다.

### ➡ DECIDE

- **2번 영역 확정** → STEP 2로 이동
- **1번·3번 영역** → 기존 스마트블록·블탭 전략 적용 (아래 참고)
- **4번 영역** → 명확한 단답형 답변 구조 집중. STEP 3~5는 동일하게 활용 가능
- **판정 불가** → 동일 키워드를 시크릿 창·모바일에서 재확인 후 재판정

#### 참고. "기존 전략" 빠른 참조

> **구조 변화 핵심:** 과거에는 블로그탭은 블로그끼리, 카페탭은 카페끼리 **따로따로** 경쟁했다. 지금의 2번 영역은 **블로그+카페+웹문서+지식인을 한 번에** 섞어서 경쟁시킨다. "한 번에"가 핵심이다. 영역 1번·3번은 이 혼합 경쟁의 영향을 덜 받으므로 기존 전략이 여전히 유효하다.

본 자료는 2번(새로운 스마트블록) 대응에 특화되어 있으나, 1번·3번 영역으로 판정될 경우 아래 정통 룰을 그대로 적용한다.

| 영역 | 적용할 기존 룰 |
|------|---------------|
| **1번. 스마트블록 + 플레이스** | • 기존 스마트블록 키워드 확장 (연관·자동완성 기반)<br>• 제목에 핵심 키워드 명시<br>• 작성자 경험 중심으로 본문 구성 |
| **3번. 인기글 (블탭+카페탭)** | • 글 시작부에 제목(키워드) 3회 반복<br>• 글 마무리부에 제목(키워드) 3회 반복<br>• 본문 키워드 반복 빈도 관리<br>• 이미지 수 확보 (장수·다양성) |

이 룰들은 새로운 스마트블록(2번)에서는 효력이 없거나 오히려 역효과가 날 수 있다. 
영역 판정이 정확해야 적용 룰을 잘못 섞지 않는다.

## STEP 2. 경쟁 분석 {#step-2}

### 💡 WHY

상위 분포를 보지 않으면, 진입 불가 영역에 글을 써서 낭비하거나 이미 블로그에 열린 영역에서 엉뚱한 구조로 쓰게 된다. 
분포는 검색자 의도의 간접 증거다. 
플랫폼 비율을 읽으면 "이 사람들이 뭘 원하는가"가 보인다.

### 🎯 ACTION

**1. 강력추천영역 상위 5~8개의 플랫폼을 기록한다.**

각 결과의 플랫폼(블로그/카페/웹문서/지식인)을 메모한다. 개수로 비율을 계산한다.

**2. 분포 해석 공식을 적용한다.**

```
카페 3개 이상   = 다수 의견으로 객관적 판단을 원하는 검색자
지식인 포함     = 질문에 대한 해답을 최우선하는 검색자
웹문서 다수     = 공식 정보·권위성 중심 탐색
블로그 다수     = 개인 경험·간접 지식 선호
```

**3. 실전 해석 예시**

| 분포 | 해석 | 블로그 진입 가능성 |
|------|------|-----------------|
| 블로그4 + 지식백과1 | 행동형·HOW 탐색 | 높음 |
| 웹문서4 + 지식인1 | 공식자료·권위성 우선 | 거의 없음 |
| 카페4 + 웹문서2 | 다수 의견 종합 + 업체 선정 | 강력추천영역 진입 불가. 6등 설계 전환 |
| 지식인3 + 카페2 | 질문+다수 의견 동시 원함 | 개인 경험 블로그로는 진입 불가 |
| 웹문서2 + 블로그3 | 전문성+경험 복합 탐색 | 있음 (전문성 명시 필수) |
| 카테고리 분류형 (드라마 추천 등) | 검색 기준 자체가 모호. AI가 단일 의도로 묶지 못해 여러 기준으로 분류해 보여주는 구조 | 전략 설계 복잡. 더 구체적인 롱테일 키워드로 전환 권장 |
| 웹문서4 + 지식인1 (학자금대출이자 등) | 공식자료·권위성 최우선. 실제로 교육부 공식 블로그가 6등에 노출됨 | 공식 블로그가 아닌 한 권위성 자체가 없음. 일반 블로그 진입 거의 불가 |

> **권위 카페:** 카페 多 분포에서 카페가 상위에 오르는 이유는 단순 다수 의견 때문만이 아니다. 운영 기간이 길고 활성도가 높은 권위 카페가 사실상의 진입 조건으로 작동한다. 권위 카페 + 공식 블로그가 상위를 점유한 키워드에서는 일반 블로그 진입을 빠르게 포기하고 6등 이하 설계로 전환한다.

**4. 상위 글 제목 패턴 7종을 추출한다.**

| 패턴 | 의미 | 내 글 제목 적용 여부 |
|------|------|---------------------|
| HOW | 어떻게 해야 하는가 | |
| WHAT | 무엇을 이야기하는가 | |
| WHO | 누구에 관한 이야기인가 | |
| WHEN | 시간·시점 명시 | |
| THAT | 비교·나열형 | |
| WHY | 이유·원인 | |
| DO | 후기·실행형 | |

상위 글 5개의 제목을 보고 어떤 패턴이 주류인지 파악한다. 
그 패턴이 검색자 의도에 가장 맞는 형식이다.

> **WHEN 패턴 주의:** "요즘", "최근", "이번 주" 같은 상대적 시간 단어가 포함된 키워드는 검색자가 시간 맥락을 갖고 검색한다. 이 경우 제목에 **구체적인 연·월·기간**을 명시한 글이 유리하다. 명시적 시간 기준이 없으면 AI가 맥락을 읽지 못해 기존 노출 글을 우선하는 경향이 있다.

**5. 상위 분포 분석을 2단계로 진행한다.**

단순히 플랫폼 비율만 보는 것으로 끝내지 않는다.

- **1단계 — 공통요소 추출 (전체 상위 1~6등):** 카페·웹문서·지식인 포함 전체 상위 글의 공통 특성을 추출한다. 
"이 영역에서 강력 추천받는 글은 어떤 공통점이 있는가"를 파악해야 소비자 의도를 정확히 읽을 수 있다.

- **2단계 — 블로그 단독 분리 분석:** 블로그만 따로 추려서 "어떤 구조·권위성·페르소나가 블로그에서 통하는가"를 분석한다. 
"홈페이지제작비용" 8등 블로그가 AI 작성 글임에도 뜬 이유가 FAQ 구조 때문임을 이 방식으로 발견했다.

**6. 상위 글 구조(FAQ vs Q&A)를 파악한다.**

| 구조 | 특징 | 유리한 조건 |
|------|------|------------|
| FAQ형 | 다양한 질문을 목록으로 모아 각각 짧게 답변 | 검색자가 여러 궁금증을 가진 경우 |
| Q&A형 | 하나의 질문에 집중해 깊이 파고듦 | 단일 질문에 명확한 답이 필요한 경우 |

### 🔍 CHECK

- 상위 5개 중 블로그가 몇 개인지 숫자로 기록됐는가?
- 주류 제목 패턴(HOW/WHAT 등) 1~2종이 확정됐는가?
- 상위 글이 FAQ형인지 Q&A형인지 판단됐는가?
- "비용이니까 비용만 써야지" 같은 협소 해석 없이 검색자의 다양한 질문군을 파악했는가?
- 웹문서 다수+지식인 키워드에서 "공식 블로그가 아닌 한 진입 불가"로 판정하고 롱테일 전환을 검토했는가?

### ⚠ WATCH OUT

- 상위 글의 구조(FAQ/Q&A)만 따라 쓰는 것으로는 부족하다. **타겟층**까지 일치해야 경쟁 대상이 된다.
- "카페가 많으니 블로그로는 1등 불가"가 아니라, 카페가 많은 키워드에서 **6등 이하 영역**을 노리는 것은 여전히 유효하다.
- 웹문서로 노출된 블로그를 "블로그 1개"로 카운트하면 판정이 틀린다.

### ➡ DECIDE

- **상위 5개 중 블로그 0개** → 강력추천영역 진입 포기. STEP 4에서 "6등 이하 설계"로 전환
- **상위 5개 중 블로그 1개 이상** → 진입 가능성 있음. STEP 3으로 이동
- **분포가 복잡하거나 카테고리 분류형** → 해당 키워드 포기 또는 더 구체적인 롱테일 키워드로 전환 후 STEP 1 재시작

## STEP 3. 페르소나 설계 {#step-3}

### 💡 WHY

페르소나 없이 쓴 글은 AI가 매칭할 상황 정보가 없다. 
결과적으로 모든 사람에게 어느 정도 맞지만, 아무에게도 딱 맞지 않는 글이 된다. 

AI는 "딱 맞는 글"을 추천하는 시스템이다. 
"어느 정도 맞는 글"은 목록에 오르지 않는다.

### 🎯 ACTION

**1. 타겟 1그룹을 확정한다. (페르소나 우선 원칙)**

같은 키워드를 검색하는 사람이라도 상황이 전혀 다르다.

예: "로봇청소기" 검색자
- 러그 없는 1인 가구 원룸 자취생
- 아이 있는 4인 가족 아파트
- 문턱 있는 구형 빌라 거주자

하나의 글로 모두를 만족시키려 하지 않는다. **한 그룹만 선택한다.**

**2. 같은 그룹 안에서 더 쪼갠다. (페르소나 세분화)**

| 키워드 예시 | 1차 타겟 | 세분화 |
|------------|---------|--------|
| 홈페이지제작비용 | 홈페이지 의뢰 기업 | 결정권 없는 말단직원 vs **결정권 있는 관리임원** |
| 서울 가볼만한 곳 | 서울 방문자 | 아이 있는 가족 vs 커플 vs 노년 부부 |
| 탄 냄비 닦는 법 | 요리 중 탄 냄비 소지자 | 급히 해결해야 하는 초보 vs 재활용 방법 탐색 |

결정권자·실무자·비전문가 중 누구를 타겟하느냐에 따라 어휘 수준, 글의 구성, 인용 출처가 달라진다.

**3. 선택한 페르소나의 5W1H + Pain Point를 도출한다.**

| 항목 | 내용 |
|------|------|
| Who | 이 사람은 누구인가 (직책·상황·경험) |
| What | 무엇을 얻으려고 검색하는가 |
| Why | 왜 지금 이 키워드를 검색하는가 |
| When | 어떤 시점·맥락에서 검색하는가 |
| Where | 어떤 환경에서 이 정보를 소비하는가 |
| How | 어떤 방식으로 문제를 해결하려는가 |
| Pain Point | 검색하게 된 근본적인 고민·불편함 |

**4. 페르소나 헬퍼를 활용한다.**

도구: `https://gpt.datalab.tools/assistant/brand/persona`

명령어 예시:
```
"[키워드]"를 검색하는 소비자 집단 5명의 5W1H와 Pain Point를 나열해줘.
```

결과에서 내가 타겟하는 1그룹만 골라 `페르소나.txt`로 저장한다.

### 🔍 CHECK

- 타겟 그룹이 1개(또는 명확히 규정된 1그룹)로 좁혀졌는가?
- 5W1H 항목이 구체적으로 채워졌는가? ("있다/없다" 수준이 아니라 상황이 그려지는 수준)
- Pain Point가 "키워드와 직접 관련된 진짜 불편함"으로 기술됐는가?

### ⚠ WATCH OUT

- "아이 있는 부모"처럼 넓은 그룹은 페르소나가 아니다. "초등학교 저학년 아이를 둔 30대 수도권 맞벌이 부부"가 페르소나다.
- 페르소나가 넓을수록 제목에 구체적인 수식어를 못 넣는다. 그 결과 파생 키워드 노출 기회도 사라진다.
- 페르소나 헬퍼 결과를 그대로 쓰지 않는다. 내가 타겟하는 1그룹을 내 판단으로 선택한다.

### ➡ DECIDE

- **페르소나 확정 완료** → STEP 4로 이동
- **페르소나가 여럿으로 좁혀지지 않음** → 다포 전략 기획. 각 페르소나마다 별도 포스팅을 계획하고, 이 회차는 1개만 선택해 진행

> **다포 전략 핵심 — 타겟 고정 함정 회피:** 연관·자동완성 검색어 기반 키워드 확장은 검색량이 큰 키워드 중심으로 이루어지므로, **타겟이 전문가에 고정된 채** 키워드 수만 늘어나는 구조다. 비전문가용 키워드에서도 전문가용 글이 작성되는 페르소나 미스매치가 발생한다. 다포 전략은 페르소나별로 키워드 집단 자체가 갈리기 때문에 이 함정을 구조적으로 우회한다.
- **페르소나와 키워드가 맞지 않음** → 키워드를 더 구체화(롱테일화)하거나 STEP 1부터 재시작

## STEP 4. 콘텐츠 설계 {#step-4}

### 💡 WHY

설계 없이 쓰기 시작하면, STEP 2에서 파악한 경쟁 구조와 STEP 3의 페르소나가 글에서 따로 논다. 제목 패턴이 검색자 의도와 어긋나거나, 경쟁 가능한 구조가 아닌 채로 발행된다. 설계 단계에서 1시간이 작성·수정 단계의 3시간을 아낀다.

### 🎯 ACTION

**1. 제목 패턴을 결정한다.**

STEP 2에서 추출한 상위 글의 주류 패턴을 참고한다. 패턴은 검색자 의도를 반영한 형식이다.

- HOW형 → "가장 빠른 방법", "단계별 가이드" 같은 수식어
- THAT형 → "A vs B", "비교해봤습니다"
- DO형 → "직접 해봤더니", "[조건] 후기"
- WHEN형 → "2026년 기준", "[월] 최신"

**제목 맥락과 본문 맥락은 반드시 일치해야 한다.** 제목에 HOW를 쓰고 본문에 WHAT만 있으면 AI가 불일치를 감지한다.

**2. FAQ vs Q&A 구조를 결정한다.**

| 선택 기준 | 판단 |
|----------|------|
| 상위 글에 FAQ형이 많고, 검색자가 다양한 궁금증을 가짐 | FAQ형 선택 |
| 상위 글에 Q&A형이 많고, 검색자가 단일 질문의 명확한 답을 원함 | Q&A형 선택 |
| 블로그끼리 경쟁하는 구간이라면 | FAQ형이 일반적으로 유리 (범위를 넓혀 더 많은 Pain Point 포괄) |

**3. 작성자 권위성 명시 방식을 결정한다.**

명시하지 않으면 AI는 기본값으로 비실무자로 가정한다.

| 권위성 유형 | 방법 |
|------------|------|
| 자기주장 권위성 | 직함·경력·전문 영역을 글 첫 부분 또는 프로필에 명시 |
| 제3자 권위성 | 공공기관·공식 기업 정보·언론 보도 등 외부 출처 인용 |

두 가지를 함께 쓸 수 있다면 더 강하다. 단, 허위 직함은 신뢰도를 역으로 훼손한다.

> **반직관 포인트:** 권위성 명시는 답을 주는 글에만 필요한 것이 아니다. "홈페이지제작비용" 1등이 기업 입장에서 기준을 알려주는 정보글이 아니라, **중소기업 기획팀 과장이 고민을 털어놓은 Q&A형 질문 글**이었다. 질문하는 글에서 작성자 직책을 밝히는 것만으로 권위성이 충분히 발현된다.

> **두 축 분리 — "잘 쓴 글" ≠ "인정받은 채널":** 새 알고리즘은 두 차원이 독립적으로 작동한다.
> - **잘 쓴 글** = 콘텐츠 품질 (페르소나·구조·작성자 권위성 명시 — 본 STEP의 영역)
> - **인정받은 채널** = 도메인 신뢰 (운영 이력·공식 인증·연결고리 5종 — STEP 6의 영역)
>
> 채널 인정이 1차 필터, 그 안에서 글 품질이 순위 결정. 따라서 "글만 잘 써도 진입 안 되는 영역"(예: 학자금대출이자)이 실재한다. 반대로 채널만 좋고 글이 부실하면 목록엔 들어가도 클릭이 없다. **두 축 모두 갖춰야 노출이 발생한다.**

> **도메인 권위:** 통합검색 진입은 사실상 도메인 권위(누가 운영하는 사이트인지, 네이버가 인정했는지)에 좌우된다. 콘텐츠 품질보다 도메인 신뢰가 먼저다. STEP 6의 연결고리 5종이 이 도메인 권위를 구축하는 방법론이다. 연결고리가 없는 상태에서 아무리 글을 잘 써도 진입 자체가 막힐 수 있다.

**4. 본문 경험 한 줄 위치를 설계한다.**

블로그는 개인 채널이다. 카페·지식인과 다른 점이 여기에 있다. 경험 한 줄은 글 초반(도입부)에 배치한다.

```
[잘못된 예] 없음. 정보 나열만.
[올바른 예] "저도 처음엔 이게 뭔지 몰라서 엄청 헤맸는데요." → 이후 정보 전개
```

공감대 형성("야 나도, 어 너도?")이 목적이다. 길 필요 없다. 단 한 줄이면 충분하다.

**5. 강력추천영역 진입 불가 시 — "6등 이하 설계"로 전환한다.**

STEP 2에서 블로그가 0개였다면 강력추천영역 진입을 포기한다. 대신:

- 강력추천영역 아래의 일반 검색 영역(블탭, 웹문서 검색)을 목표로 설정
- 제목·구조는 동일하게 설계하되, "1등을 노리는 글"이 아니라 "검색하면 나오는 글"로 목표 조정
- 파생 롱테일 키워드를 동시에 공략해 노출면 확대

### 🔍 CHECK

- 제목 패턴(HOW/WHAT 등)이 확정되고, 실제 제목 초안이 1개 이상 만들어졌는가?
- 제목에 페르소나가 검색할 키워드(주어)가 명확히 포함됐는가?
- FAQ/Q&A 구조가 결정됐는가?
- 작성자 권위성 명시 위치가 정해졌는가?
- 경험 한 줄 삽입 위치가 도입부로 계획됐는가?

### ⚠ WATCH OUT

- "내가 쓰고 싶은 내용"이 아니라 "페르소나가 필요로 하는 내용"으로 본문 항목을 구성한다. 이 둘이 다를 때 항상 페르소나 기준을 따른다.
- FAQ형을 선택했다면 항목 하나하나가 "비용만", "기간만" 같은 단일 주제로 좁혀지면 안 된다. 페르소나의 Pain Point 전체를 포괄하는 항목군을 설계한다.
- 작성자 권위성이 없는 상태에서 전문가 어조로만 쓰면 오히려 역효과다.

### ➡ DECIDE

- **설계 완료** → STEP 5로 이동
- **제목 초안이 5개 이상 나왔는데 다 애매함** → 페르소나를 더 좁히거나 구체화 후 재설계
- **6등 이하 설계로 전환** → STEP 5는 동일하게 진행. STEP 6에서 발행 후 모니터링 기준을 조정

## STEP 5. 작성·교정 {#step-5}

### 💡 WHY

AI 도구로 초벌을 뽑는 것과 발행 가능한 글을 만드는 것은 다르다. AI 생성 글은 사실 오류, 맥락 이탈, 소설 같은 서술이 섞여 있다. 교정 없이 그대로 발행하면 권위성이 역으로 훼손된다. 사람의 교정이 반드시 끼어야 하는 이유다.

### 🎯 ACTION

**7단계 AI 워크플로 (압축형)**

| 단계 | 도구 | 핵심 명령 |
|------|------|----------|
| 1. 페르소나 도출 | 페르소나 헬퍼 | `"[키워드]"를 검색하는 소비자 집단 5명의 5W1H와 Pain Point를 나열해줘.` |
| 2. 핵심 초안 직접 작성 | 직접 (사람이) | 내가 경험한 핵심 사실만, 200자 수준 |
| 3. 1차 초벌 생성 | 톤앤매너 헬퍼 | `초안의 톤앤매너를 유지하며, 페르소나의 5W1H를 충족하는 1500자 블로그 글 작성. Pain Point에 대한 공감·꿀팁 포함.` |
| 3-1. 사람 교정 | 직접 (사람이) | 사실 오류 수정. 교정안.txt로 저장. |
| 4. 2차 교정 | 초안작성 헬퍼 | `페르소나의 5W1H와 Pain Point를 완전하게 충족하도록 교정안을 재교정` |
| 5. 최적화 | 검색최적 헬퍼 | 동일 명령 후 → `[타겟 페르소나]에 집중해서 다시 작성` |
| 6. (선택) 구글 대응 | 검색최적 헬퍼 | `SEO/AEO/GEO에 정합하도록 재작성` |

**데렙툴즈 도구 URL**

| 도구 | URL |
|------|-----|
| 페르소나 헬퍼 | `https://gpt.datalab.tools/assistant/brand/persona` |
| 톤앤매너 헬퍼 | `https://gpt.datalab.tools/assistant/brand/voice` |
| 초안작성 헬퍼 | `https://gpt.datalab.tools/assistant/creator/write` |
| 검색최적 헬퍼 | `https://gpt.datalab.tools/assistant/marketing/saeo` |

**키워드(주어) 사용 원칙**

AI는 글에서 "무엇에 대한 글인지"를 주어(키워드)로 파악한다. 대명사("이거", "저거", "그것", "이 제품")로 교체하면 AI가 맥락을 읽지 못한다.

```
[잘못된 예] "이걸 사용해보니 생각보다 훨씬 괜찮더라고요. 그게 문제가 됐을 때..."
[올바른 예] "로봇청소기를 사용해보니 생각보다 훨씬 괜찮더라고요. 로봇청소기가 문턱에서 막혔을 때..."
```

자연스러운 범위 내에서 키워드(주어)를 유지한다. 무리한 반복은 필요 없다. 대명사로 **대체하지 않는** 것이 핵심이다.

**최종 글에서 확인해야 할 3가지**

1. 페르소나 한 그룹에 집중되어 있는가? (모든 사람을 만족시키려는 글이면 탈락)
2. 경험 한 줄이 도입부에 들어갔는가?
3. 제목의 맥락과 본문의 맥락이 일치하는가?

### 🔍 CHECK

- 2단계(핵심 초안)는 사람이 직접 썼는가? AI 생성 초안을 넣지 않았는가?
- 3-1단계 사람 교정이 실행됐는가? 사실 오류가 제거됐는가?
- 최종 글의 키워드가 대명사로 교체되지 않고 자연스럽게 유지됐는가?
- 작성자 권위성(직함·경력)이 글에 명시됐는가?
- 경험 한 줄이 도입부에 삽입됐는가?

### ⚠ WATCH OUT

- AI 도구를 여러 단계 거칠수록 내 말투가 희석된다. 단계마다 톤이 맞는지 확인한다.
- "AI가 썼는데 왜 교정해야 하나"는 잘못된 사고다. AI는 사실을 모른다. 내가 경험한 내용과 다른 내용을 그럴듯하게 생성한다.
- 최적화 단계(5·6단계) 이후 글이 많이 바뀌었다면 반드시 다시 읽고 페르소나 집중도를 확인한다.

### ➡ DECIDE

- **최종 글 완성** → STEP 6으로 이동
- **글이 여러 타겟을 동시에 커버하려 함** → 페르소나 집중도 점검 후 재교정
- **경험 한 줄이 들어가기 어려운 분야(법률·의료 등)** → "저도 이 문제로 고민했을 때 가장 먼저 확인한 게..." 같은 간접 공감 형식으로 대체

## STEP 6. 발행·운영 {#step-6}

### 💡 WHY

외부링크 하나가 잘못 삽입되면 글 전체가 노출 차단된다. 
차단은 URL 단위가 아니라 도메인 전체 단위로 이루어진다. 
블로그가 정상이어도 링크 삽입된 글만 안 보이거나, 글과 함께 블로그 노출 자체가 영향을 받는다. 
발행 전 30초 확인이 며칠치 작업을 날리는 상황을 막는다.

### 🎯 ACTION

**1. 외부링크 삽입 전 "도메인 검색 1등" 단순 판단법 적용**

```
판단 1: [도메인 이름]을 네이버에서 검색했을 때 대표사이트가 1등에 뜨는가?
  → 안 뜨면 사용하지 않는다.

판단 2: 검색결과에 "관련문서 더보기"가 딸려 나오는가?
  → 없으면 정보가치 없는 사이트로 분류. 삽입 시 스팸 위험.
```

1페이지짜리 랜딩 페이지, 단순 리디렉션 링크, 대표사이트가 뜨지 않는 도메인은 삽입하지 않는다.

**2. 외부링크 안전 게재를 위한 연결고리 5종 점검**

| 연결고리 | 점검 내용 |
|----------|----------|
| 네이버 ID | 블로그·광고·웹마스터 도구가 동일 계정으로 연결됐는가 |
| 사업자정보 | 블로그 사업자정보에 홈페이지 등록됐는가 |
| 홈페이지 도메인 | 네이버 웹마스터 도구에 등록하고 소유주 확인 완료됐는가 |
| 광고 이력 | 네이버 브랜드 광고 집행 이력이 있는가 (정상 브랜드 검수 효과) |
| 인물정보 | 개인 블로그라면 인물정보 등록으로 블로그와 1:1 연결됐는가 |

연결고리가 많을수록 "이거 다 내 거야"의 법적 근거가 두터워진다. 최소 1·2·4번은 확보한다.

**3. URL 구조 안전 판단**

| URL 형식 | 스팸 위험도 |
|----------|------------|
| `https://domain.com/` (메인 도메인) | 없음. 가장 안전 |
| `/카테고리/글번호` (sub path) | 있음. 동일 경로 반복 시 스팸 |
| `/index.php?id=12345` (params 방식) | 상대적으로 낮음. 1경로 다역할 구조 |
| 100% 동일한 URL 반복 삽입 | 무조건 스팸 처리 |

메인 도메인 링크 사용을 우선한다.

> **참고 (정론 아님) — 티스토리 우회 구조:** 티스토리로 공사치는 블로그의 스팸 회피 구조는 이중 엮음이다. ① 티스토리 글들끼리 내부 링크로 빽빽이 엮고, ② 티스토리에서 외부 사이트로 링크를 연결한다. 네이버 블로그에는 매번 새 링크를 생성해 100% 동일 URL 반복을 피한다. 이 구조 자체가 스팸 감지를 우회하기 위한 꼼수임을 인지해두는 수준으로만 참고한다.

**4. 댓글·톡톡 우회법 적용 (외부 유도가 필요한 경우)**

- **댓글 링크 유도**: 댓글 스팸 처리는 글 노출에 영향을 주지 않는다. 
본문에 외부링크를 직접 삽입하는 대신 "댓글로 공유"하는 방식이 더 안전하다.

- **네이버 톡톡**: 블로그 스마트에디터에 톡톡 버튼 삽입. 
이미지 크기 무제한, 네이버 공식 기능이므로 스팸 문제 없음. 
클릭 시 1:1 상담 연결. 단, 외부 사이트·오픈채팅으로의 직접 유도는 불가.

**5. 발행 후 노출 모니터링**

발행 후 24~48시간 내 확인:

- 해당 키워드를 시크릿 창에서 검색해서 글이 뜨는지 확인
- 강력추천영역에 들어갔는지, 6등 이하 영역인지 위치 기록
- 노출 없을 경우: 블로그 사업자정보·웹마스터 도구 등록 상태 재점검

### 🔍 CHECK

- 삽입한 외부링크의 도메인을 네이버에서 검색했을 때 1등에 대표사이트가 뜨는가?
- 동일 URL을 다른 글에서도 반복 삽입하지 않았는가?
- 연결고리 5종 중 최소 네이버 ID·사업자정보·도메인 3가지는 연결됐는가?
- 발행 후 48시간 이내에 노출 여부를 확인했는가?

### ⚠ WATCH OUT

- 외부링크 차단은 URL 단위가 아니라 **도메인 전체 단위**다. 하위 페이지 링크가 막히면 그 도메인의 모든 링크가 차단된다.
- KISA(인터넷진흥원) 신고가 접수되면 합법 사이트라도 계정 전체에 영향이 간다. 피싱·불법 여부와 무관하다.
- 네이버 인플루언서·클립·프리미엄콘텐츠는 동일 계정이면 2022년 이후 문제없이 삽입 가능하다. 이를 외부링크와 혼동하지 않는다.

### ➡ DECIDE

- **외부링크 없이 발행** → 연결고리 점검 생략 가능. 바로 발행
- **외부링크 있음 + 연결고리 미흡** → 댓글 유도 방식으로 대체하거나, 연결고리 구성 후 삽입
- **발행 후 48시간 이상 미노출** → 키워드 재진단(STEP 1 재시작) 또는 글 구조·페르소나 재점검

## 부록 2. 자주 묻는 판단 케이스 {#appendix-2}

**Q. 키워드를 받았는데 상위 5개에 블로그가 한 개도 없다.**

A. 강력추천영역 진입을 포기한다. 해당 영역은 카페·웹문서·지식인이 구조적으로 유리하도록 AI가 판단한 결과다. 블로그로 1등을 만들려면 대규모 트래픽 조작이나 수십만 개의 배포 블로그가 필요하다. 현실적인 대안은 두 가지다. 첫째, 강력추천영역 아래의 6등 이하 영역을 노린다. 둘째, 더 구체적인 롱테일 키워드로 전환해 STEP 1부터 재시작한다.

---

**Q. 카페가 상위에 많은데 일반 블로그로 들어갈 여지가 있나?**

A. 카페 多 키워드에서 상위를 차지하는 카페가 권위 카페(오래되고 활성도 높은 대형 카페)라면 일반 블로그로 강력추천영역 진입은 사실상 불가다. 권위 카페 + 공식 블로그가 자리를 점유한 구조이기 때문이다. 이 경우 강력추천영역 진입을 포기하고 ① 6등 이하 영역 노리기 또는 ② 더 구체적인 롱테일 키워드로 전환하는 것이 현실적이다. 카페의 대화·질문이 다양하다면 FAQ형 구조로 블로그 글을 작성해 6등 이하 노출을 목표로 한다.

---

**Q. 상위 5개 분석을 했는데 플랫폼이 섞여서 판단이 어렵다.**

A. 분포 해석 공식의 "가장 많은 플랫폼" 기준으로 주 의도를 파악한다. 카페 2개, 블로그 2개, 웹문서 1개라면 카페와 블로그가 혼재하는 구간이다. 이 경우 블로그 진입 가능성은 있으나, 카페 특성(다수 의견 종합)을 고려해 FAQ형 구조가 유리하다. 판단이 여전히 어렵다면 상위 블로그 글의 구조를 직접 읽고 벤치마킹한다.

---

**Q. 페르소나를 여러 개 설정하고 싶다. 하나로 좁혀야 하나?**

A. 1개의 글에는 1개의 페르소나만 적용한다. 여러 페르소나를 동시에 만족시키려는 글은 아무도 만족시키지 못한다. 대신 페르소나별로 글을 따로 발행하는 다포 전략을 사용한다. 같은 주제에서 타겟별로 다른 키워드 집단이 만들어지므로 포스팅 수가 늘어날수록 노출면도 넓어진다.

---

**Q. 작성자 권위성이 없다. 전문가가 아닌데 어떻게 권위성을 명시하나?**

A. 직함이 없어도 경험이 권위성이 된다. "3년째 로봇청소기를 바꿔온 1인 가구 자취생"이 "전문가" 없이도 해당 페르소나에게는 충분한 권위성이다. 중요한 것은 **명시**다. 아무것도 밝히지 않으면 AI는 비실무자로 가정한다. 제3자 권위성(공공기관·공식 통계 인용)을 보강하면 개인 권위성의 한계를 일부 커버할 수 있다.

---

**Q. 외부링크를 넣고 싶은데 내 홈페이지 도메인이 네이버 검색 1등이 아니다.**

A. 홈페이지 링크를 본문에 직접 삽입하지 않는다. 댓글 유도 방식(본문에서 "댓글로 공유합니다" 안내 + 댓글에 링크 삽입)을 사용한다. 댓글 스팸 삭제는 글 노출에 영향을 주지 않는다. 또는 네이버 톡톡을 활용해 1:1 상담 경로를 만들고 그 과정에서 링크를 전달한다.

---

**Q. AI 브리핑 영역에 들어가려면 어떻게 써야 하나?**

A. "~방법", "~이유", "~원인", "~차이" 같은 질문-답변이 1:1로 떨어지는 키워드가 AI 브리핑 대상이다. 이 영역은 명확한 단답 구조가 핵심이다. 질문을 제목이나 소제목으로 명시하고, 바로 아래에 명확한 답변을 배치하는 Q&A형 구조가 유리하다. 근거(출처·데이터)를 함께 제시하면 AEO 관점에서 유리하다.

---

**Q. 글을 발행했는데 48시간이 지나도 뜨지 않는다.**

A. 순서대로 점검한다. 첫째, 블로그 자체가 새 블로그라면 초기 인덱싱에 시간이 더 걸릴 수 있다. 둘째, 웹마스터 도구에서 블로그가 등록됐는지 확인한다. 셋째, 삽입한 외부링크 중 스팸으로 감지됐을 가능성이 있는 도메인이 있다면 제거 후 재발행한다. 넷째, STEP 1로 돌아가 키워드 영역 판정이 맞았는지 재확인한다. 진입 가능성이 낮은 영역이었다면 키워드 자체를 변경한다.
