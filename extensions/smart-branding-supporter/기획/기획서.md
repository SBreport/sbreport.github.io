# 스마트 브랜딩 서포터 (Smart Branding Supporter, SBS) 기획서

- **작성일**: 2026-05-15
- **버전**: v0.1 초안
- **한 줄 요약**: 네이버 블로그 운영·검색 분석에 필요한 4개 핵심 기능을 하나의 Chrome 확장으로 통합하여, 외부 도구 의존성을 제거하고 사내 자산으로 확보한다.

---

## 1. 프로젝트 개요

### 배경

기존에는 외부 Chrome 확장에 의존해 검색량 조회·네비게이션·블로그 강조 등을 수행해왔다. 그러나 주요 참고 도구의 백엔드 운영이 중단되었고, 외부 도구는 유지보수·보안·지속성 모두에서 리스크를 안는다. 사내에 유일하게 보유한 확장 자산(`naver-blog-cleaner`)은 글감 패널 숨기기 단일 기능에 그친다.

이러한 상황에서 4개 핵심 기능을 하나의 확장에 통합하고, 코드는 클린룸 방식으로 자체 작성하여 사내 자산화한다. 인수인계 가능한 구조와 명확한 식별자 체계를 갖추는 것이 설계 원칙의 출발점이다.

### 목표

| 항목 | 내용 |
|------|------|
| 기능 통합 | 4개 기능을 단일 확장으로 (글감 숨기기 / 네비게이터 / 블로그 강조 / 검색량 박스) |
| 외부 의존 최소화 | 데이터 출처는 네이버 공식 API(검색광고)만 사용 |
| 클린룸 구현 | 외부 도구 코드 직접 이식 금지. 패턴·UX만 참고, 자체 작성 |
| 사내 자산화 | MIT 라이선스, 명확한 폴더 구조, 인수인계 가능한 문서 |

## 2. 핵심 기능

### 기능 ①: 블로그 글감 패널 숨기기

- **목적**: 블로그 에디터 우측 글감 패널을 토글로 숨겨 집중력 확보
- **적용 페이지**: `blog.naver.com`, `editor.*.naver.com`, `m.blog.naver.com`
- **동작**: 확장 아이콘 클릭 → 숨김/표시 토글
- **자산 출처**: 기존 `naver-blog-cleaner` 코드 이식 (셀렉터·MutationObserver 패턴 검증됨)
- **데이터 출처**: 없음 (DOM 조작만)
- **우선순위**: P1 — 가장 단순, Phase 1에서 먼저 완성

---

### 기능 ②: 네이버 검색 결과 좌측 네비게이터

- **목적**: 검색 결과 페이지의 섹션 목록을 좌측 패널로 띄우고, 클릭 시 해당 섹션으로 부드럽게 스크롤
- **적용 페이지**: `search.naver.com/search.naver`
- **동작**:
  - 페이지 진입 시 자동 활성화
  - 스크롤스파이: 현재 뷰포트에 보이는 섹션 자동 강조
  - UGC 섹션(블로그/카페/지식iN/인플루언서)은 녹색 톤, 그 외(뉴스/이미지/동영상/쇼핑/플레이스 등)는 회색 톤
  - 우측 상단 닫기 버튼 제공

- **기술 핵심**:

  | 항목 | 내용 |
  |------|------|
  | 섹션 인덱스 출처 | 네이버 전역 변수 `nx_cr_area_info` (안정적, 네이버가 직접 박아둔 값) |
  | 폴백 셀렉터 체인 | `[data-meta-area^=ID]` → `[data-slog-container^=ID]` 순서로 탐색 |
  | 패널 위치 추적 | `requestAnimationFrame` + 8% interpolation (부드러운 follow) |
  | 스크롤 동작 | `scrollTo({ behavior: "smooth" })` + 상단 70px 여유 |
  | 검색어 변경 감지 | URL 변경 감지 후 섹션 목록 재파싱 |

- **데이터 출처**: 페이지 HTML 자체 (외부 API 호출 없음)
- **우선순위**: P2 — Phase 2

---

### 기능 ③: 블로그 항목 녹색 박스 강조

- **목적**: 검색 결과 카드 중 블로그 출처인 항목만 시각적으로 강조하여 블로그 마케팅 분석 효율화
- **적용 페이지**: `search.naver.com/search.naver`
- **동작**:
  - 페이지 진입 시 자동 활성화
  - 블로그 카드 컨테이너에 `outline: 3px solid #2db400` 적용
  - 무한스크롤로 추가 로드된 결과도 즉시 반영

- **기술 핵심**:

  | 항목 | 내용 |
  |------|------|
  | 카드 셀렉터 | `.fds-ugc-block-mod` / `[data-template-id="ugcItem"]` / `[data-template-id="ugcItemDesk"]` 3종 병행 |
  | 블로그 판정 | 카드 내부 본문 링크 추출 → URL 도메인이 `blog.naver.com` / `m.blog.naver.com` 이면 블로그로 간주 |
  | URL 정규화 | 자체 유틸 `lib/service-url-parser.js` 사용 (URL → `{serviceType, serviceId}`) |
  | 동적 결과 대응 | `MutationObserver`로 DOM 변경 감지 후 즉시 판정·강조 |

- **데이터 출처**: 페이지 DOM (외부 API 호출 없음)
- **우선순위**: P2 — Phase 3

---

### 기능 ④: 검색량 박스 표시

- **목적**: 검색창 우측에 PC/모바일/총 검색량을 실시간으로 노출하여 마케팅 핵심 지표 즉시 확인
- **적용 페이지**: `search.naver.com/search.naver`
- **동작**:
  - 검색어가 바뀌면 자동으로 새 검색량 조회
  - 검색창 우측에 인라인 박스 표시 (닫기 버튼 포함)
  - 동일 키워드 24시간 캐시 (`chrome.storage.local`)

- **기술 핵심**:

  | 항목 | 내용 |
  |------|------|
  | 데이터 출처 | 네이버 검색광고 API (`https://api.naver.com/keywordstool`) |
  | 응답 필드 | `monthlyPcQcCnt`, `monthlyMobileQcCnt` |
  | 인증 방식 | HMAC-SHA256 Signature (Access License + Secret Key + Customer ID) |
  | 보안 | **클라이언트 직접 호출 금지** — Secret Key 노출 위험. 자체 백엔드 프록시 경유 |
  | 백엔드 | Cloudflare Workers (무료 티어, 월 10만 req/일) |
  | 캐시 | `chrome.storage.local` 24시간 (선택적으로 Workers KV 추가) |

- **의존**: 네이버 검색광고 계정 + API Key 발급 + Cloudflare Workers 셋업 (후행 조건)
- **우선순위**: P3 — Phase 4 (백엔드 셋업 완료 후 착수)

## 3. 기술 스택

| 레이어 | 선택 | 비고 |
|--------|------|------|
| 확장 매니페스트 | MV3 | Chrome 표준 최신 스펙 |
| Content Scripts | Vanilla JS | 프레임워크 없음, 외부 의존성 0 |
| Popup UI | HTML + CSS + Vanilla JS | 경량 유지 |
| 백엔드 | Cloudflare Workers | 무료 티어 (월 10만 req/일 한도) |
| 백엔드 언어 | JavaScript | Workers 표준 환경 |
| 외부 API | 네이버 검색광고 API | `keywordstool` 엔드포인트만 사용 |
| 클라이언트 캐시 | `chrome.storage.local` | 24시간 TTL |
| 서버 캐시 | Workers KV | 선택적 도입 (비용 발생 여부 TBD) |
| 빌드 도구 | 없음 | 소스 = 배포. 빌드 파이프라인 불필요 |
| 코드 스타일 | 글로벌 CLAUDE.md 원칙 | 정보 밀도, AI Slop 회피, 간결한 DOM 조작 |

## 5. 클린룸 방침

외부 확장(외부 도구들)의 코드를 직접 이식하지 않는다. UX 패턴·DOM 셀렉터 전략·데이터 흐름만 참고하고, 코드는 전량 자체 작성한다.

**식별자 규칙**

| 대상 | 규칙 | 예시 |
|------|------|------|
| DOM ID / CSS 클래스 | `sbs-` 접두사 | `sbs-navigator`, `sbs-volume-box` |
| Chrome 메시지 타입 | `SBS_` 접두사 | `SBS_FETCH_VOLUME`, `SBS_TOGGLE_NAV` |
| 외부 도구 명칭 | 코드·주석·문서 전면 금지 | — |
| 라이선스 | MIT | 자체 자산 |

## 8. 참고: 이전 외부 도구 분석 요약

**외부 도구 A (운영 중단)**: 검색량 박스 표시 및 좌측 섹션 네비게이터의 UX 패턴을 참고했다. 자체 백엔드 서버가 폐쇄되면서 기능이 전면 중단되었으며, 이 사례가 외부 도구 의존 탈피의 직접적 계기가 되었다. 해당 도구와의 결별 의지는 명확하다.

**외부 도구 B (운영 중)**: 검색 결과 카드 셀렉터(`.fds-ugc-block-mod`, `[data-template-id="ugcItem(Desk)"]`) 및 블로그 식별 로직(URL 도메인 매칭)의 전략을 참고했다. 코드를 직접 인용하지 않으며, 패턴과 셀렉터 구조만 학습했다.

**사내 `naver-blog-cleaner`**: 기능 ①의 기반이 되는 자산이다. 글감 컨테이너 셀렉터 및 MutationObserver 패턴이 실사용 환경에서 검증된 상태이며, 식별자 체계(`sbs-` 접두사)로 변환 후 이식할 예정이다.

## 9. 진행 방식

이 문서는 세션 간 공유 기준점이다. 다음 원칙으로 운영한다.

- 사용자가 "기능 ① 진행하자"고 하면 해당 챕터만 집중 논의한다. 전체를 한꺼번에 다루지 않는다.
- 매 Phase 종료 시 이 문서의 체크리스트를 업데이트한다.
- 새롭게 결정된 사항은 7번 섹션의 해당 행을 수정하거나 완료 표시한다.
- 이 문서를 펼치는 것만으로 이전 세션의 맥락을 복원하고 즉시 작업 재개할 수 있어야 한다.
