## PART 0. 사전 준비 — 용어 정리 {#part-0}

> 💼 **마케터/대행사 관점 활용법:** 이 파트의 용어를 숙지해두면 클라이언트 보고서나 내부 전략 회의에서 정확한 언어로 소통할 수 있습니다.

강의 전체에서 자주 등장하는 핵심 용어를 먼저 정리합니다. 모르는 단어가 나오면 이 표로 돌아오세요.

| 용어 | 풀이 |
|------|------|
| **스마트블록** | 네이버 통합검색에서 카테고리별로 묶여 나오는 검색결과 묶음. 블로그·카페·웹문서 등이 각자 경쟁 후 섞여 노출됨 |
| **블탭 (블로그탭)** | 네이버 검색결과의 '블로그' 탭. 블로그끼리만 경쟁하는 구역 |
| **최적블** | 블로그탭에서 최적화된 블로그. 기존 블로그 지수 기반 노출 방식 |
| **통검** | 통합검색. 블로그·카페·웹문서·지식인·뉴스 등이 한 화면에 섞인 결과 |
| **롱테일 키워드** | 검색량이 적지만 구체적인 키워드. 예: "서울 홍대 근처 조용한 브런치 카페" |
| **AI 브리핑** | 질문 하나에 답변 하나가 명확히 매칭되는 키워드에서 상단에 뜨는 AI 요약 답변 |
| **Pain Point** | 검색자가 그 키워드를 검색하게 된 근본적인 고민·불편함·이유 |
| **5W1H** | Who(누가), What(무엇을), Why(왜), When(언제), Where(어디서), How(어떻게). 데이터분석·프로파일링 표준 용어. 우리말 '육하원칙'보다 분석 맥락에서 씀 |
| **SEO** | Search Engine Optimization. 검색엔진 최적화. 흔히 말하는 '상위노출 로직' |
| **AEO** | Answer Engine Optimization. 답변엔진 최적화. 사용자 질문에 명확한 해답 제시 |
| **GEO** | Generative Engine Optimization. AI가 내 글을 인용해 답변을 생성하고 출처를 표기하게 만드는 기술 |
| **페르소나** | 내 글을 읽을 대상 독자를 구체적으로 설정한 가상 인물상 |
| **데렙툴즈** | 데이터랩 도구 모음 사이트 (gpt.datalab.tools). ChatGPT와 연동된 블로그 작성 보조 도구 |
| **톤앤매너** | 글의 말투·분위기·스타일 |
| **FAQ** | Frequently Asked Questions. 자주 묻는 질문 목록을 모아 각각 짧게 답변하는 형식 |
| **Q&A** | 하나의 질문에 깊이 파고드는 심층 답변 형식 |
| **권위성** | 작성자 또는 출처가 얼마나 신뢰·전문성을 인정받는가의 지표 |

## PART 1. 이해 — 네이버는 왜, 어떻게 바뀌었나 {#part-1}

> 💼 **마케터/대행사 관점 활용법:** 클라이언트가 "예전 방식으로 해주세요"라고 요청할 때, 검색 구조가 바뀐 이유를 논리적으로 설명할 근거가 됩니다.

### 1.1 네이버가 AI 검색으로 바뀌는 이유

네이버가 AI를 도입한 이유는 단순합니다. 사용자가 원하는 정보를 더 빠르게, 더 정확하게 **떠먹여주기** 위해서입니다.

예전에는 검색결과 순위를 사람이 설정한 고정 기준으로 정렬했습니다. 지수를 쌓고, 키워드를 반복하고, 텍스트 분량을 채우는 방식이 먹혔죠. 하지만 AI는 다릅니다. 사용자의 과거 검색·클릭 이력을 분석해 **그 사람이 지금 어떤 상황인지, 무엇을 찾는지** 실시간으로 파악합니다.

또한 AI는 변화 반영 속도가 압도적으로 빠릅니다. 예전에는 새로운 검색 트렌드가 생기면 통계를 쌓고 A/B 테스트를 거쳐야 해서 수주일이 걸렸습니다. AI는 변경된 부분만 국소적으로 재해석해 적용하기 때문에 단 몇 시간~하루 이틀 안에 반영됩니다.

### 1.2 네이버 검색결과 4종 분류

현재 네이버 통합검색 결과는 크게 4가지로 나뉩니다.

| 유형 | 특징 | 기존 전략 적용 여부 |
|------|------|---------------------|
| **1. 스마트블록 + 플레이스** | 장소·지역 키워드 중심 | 기존 스마트블록 전략 그대로 유효 |
| **2. 새로운 스마트블록 (AI 기반 순위 재정렬)** | 블로그+카페+웹문서+지식인 혼재. AI가 소비자 맥락에 따라 순위 결정 | 새로운 접근 필요 (이 강의의 핵심) |
| **3. 인기글 (블탭+카페탭)** | 블로그탭·카페탭 각각의 탭 노출 | 기존 블탭 전략 그대로 유효 |
| **4. AI 브리핑** | "~방법", "~원인", "~이유" 같이 질문-답변이 1:1로 떨어지는 키워드에 상단 요약 박스로 등장 | 명확한 답변 제공에 집중 |
| **5. 쇼핑 광고 + 스마트블록 혼합형** | 상단 쇼핑 광고 + 관련 정보글 + 네이버 스토어 상품이 함께 노출되는 패턴. 예: "탄 냄비 닦는 법" — 지식백과+블로그+네이버스토어. 그 아래로 내려가면 새로운 스마트블록이 추가 배치됨 | 상품리뷰·쇼핑 연계 키워드에서 자주 발생. 블로그 정보글 + 구매 연결을 동시에 설계 |

> 유명 강사들의 유튜브 강의가 틀린 게 아닙니다. 1번·3번 영역은 기존 전략이 여전히 먹힙니다. 이 강의가 다루는 건 주로 **2번 영역** 입니다.

#### 키워드 명명 형태별 영역 분기 — 30초 예측법

같은 도메인의 키워드라도 명명 형태에 따라 노출 화면이 달라집니다. 키워드를 받으면 **형태부터 보고 어느 영역에 떨어질지 먼저 예측** 하세요.

| 명명 형태 | 예시 | 노출 영역 | 이유 |
|----------|------|----------|------|
| **이름형** (브랜드명·병원명) | ○○치과, ○○클리닉 | 통합검색 (브랜드 검색) | 특정 브랜드를 찾는 지명 검색으로 처리됨 |
| **행위형** (시술명·서비스명) | ○○ 보톡스, ○○ 시술 | 인기글 화면 (정보 탐색) | AI가 정보 탐색 의도로 분류. 리뷰·후기 글 중심 노출 |

이름형 키워드에서 스마트블록 전략을 쓰거나, 행위형 키워드에서 브랜드 페이지만 연결하면 영역 불일치로 노출이 어긋납니다. **영역 예측 → 전략 선택** 순서를 지키세요.

#### "기존 전략"의 구체 내용

표에서 "기존 전략 그대로 유효"라고 표기한 1번·3번 영역에서 의미하는 기존 전략의 실체는 다음과 같습니다.

| 영역 | 기존 전략 |
|------|-----------|
| **1번. 스마트블록 + 플레이스** | • 연관·자동완성 기반 키워드 확장<br>• 제목에 핵심 키워드 명시<br>• 작성자 직접 경험 중심 본문 구성 |
| **3번. 인기글 (블탭+카페탭)** | • 글 시작부에 제목(키워드) 3회 반복<br>• 글 마무리부에 제목(키워드) 3회 반복<br>• 본문 키워드 반복 빈도 관리<br>• 이미지 수 확보 (장수·다양성) |

이 룰은 2번 영역(새로운 스마트블록)에서는 효력이 없거나 오히려 역효과가 날 수 있습니다. 영역 판정이 정확해야 적용 룰이 어긋나지 않습니다.

### 1.3 검색결과 렌더링 패턴 식별법 ⭐

검색결과를 보면서 "여기가 어떤 구역인지"를 빠르게 파악하는 방법입니다.

#### "더보기" 3가지 패턴

| 패턴 | 생김새 | 의미 |
|------|--------|------|
| **강력추천영역** | 5~8개 결과 묶음 + "더보기" 없음 (묶음만 덩그러니) | 새로운 스마트블록의 핵심 노출 구역 |
| **웹문서 검색** | 5~8개 결과 묶음 + 묶음 아래 "더보기" 버튼 존재. 닉네임 대신 도메인(blog.naver.com/아이디) 표시 | 웹문서로 노출된 것. 일반 블로그 노출과 구분 필요 |
| **페이지번호형** | "더보기" 없이 1/2/3/4/5... 페이지 번호 | 블로그+지식인 혼재 영역 |

> 중요: `blog.naver.com`처럼 닉네임 자리에 도메인과 아이디가 회색으로 표시되면 그것은 **웹문서로 노출된 것** 입니다. 블로그 노출과 따로 분석해야 합니다.

#### 17종 조합 / 5~8개 가변

네이버의 검색결과 렌더링 방식은 조합 기준으로 17종에 달합니다. 결과 수도 최소 5개에서 최대 8개까지 가변적입니다. 단순히 "5개인지 6개인지"로 판단하지 말고, 플랫폼 유형(블로그/카페/웹문서/지식인)과 더보기 유무로 구역을 먼저 파악하세요.

### 1.4 플랫폼끼리 따로 경쟁 후 섞이는 구조

과거에는 블로그탭은 블로그끼리, 카페탭은 카페끼리, 웹문서탭은 웹문서끼리만 **따로따로** 경쟁했습니다. 지금은 다릅니다. **블로그+카페+웹문서+지식인을 한 번에 섞어서 경쟁** 시킨 뒤, 그 결과를 **소비자 맥락과 선호도에 따라 일정 비율로 섞어** 통합검색에 보여줍니다. "한 번에"가 핵심입니다. 플랫폼 간 장벽이 사라진 구조입니다.

- 카페는 카페끼리 경쟁 → 추천도 순 정렬 → 일부가 통합검색에 올라옴
- 블로그는 블로그끼리 경쟁 → 추천도 순 정렬 → 일부가 통합검색에 올라옴
- 웹문서는 웹문서끼리 경쟁 → 일부가 통합검색에 올라옴

이 구조를 이해하면, 상위 5개에 블로그가 한 개도 없는 키워드에서 "어떻게 하면 블로그로 1등 할 수 있냐"는 질문 자체가 잘못됐다는 걸 알 수 있습니다.

### 1.5 AI 검색의 본질 — "AI는 중매쟁이다"

> "AI는 일종의 중매쟁이에요. 검색하는 사람들은 몰라서, 또는 더 알고 싶어서 찾는 거고. 그 사람들은 어떤 공통된 상황과 목적을 가지고 있어요. AI는 그 맥락을 파악해서 맞는 글을 '추천'해주는 거에요."

결혼정보회사(결정사) 같다고 생각하면 됩니다.

1. 검색자: "이런 조건이고 저런 상황인데 뭔가 없나?" 하고 검색
2. AI(중매쟁이): 검색자의 과거 행동 패턴 분석 → 소구점 도출 → 조건에 맞는 글 필터링
3. 내 글(배우자 후보): AI가 샅샅이 프로파일링해서 매칭 가능 여부 판단
4. 결과: "3명 정도 매칭되시네요 씩" → 검색결과 노출

핵심: **1등이 목표가 아니라, 일단 매칭 목록에 올라가는 것이 첫 관문** 입니다.

### 1.6 왜 1등이 아니라 "목록 진입"이 먼저인가

AI가 소비자 맥락과 내 글을 매칭할 때, 내 글이 얼마나 "그 사람의 상황에 딱 맞는가"가 기준입니다. 1등을 노리기 전에 먼저 목록에 올라야 클릭이라도 받을 수 있습니다.

비유하자면: 데이트 앱에서 프로필을 아무도 안 보는 상태에서 "1등 추천받겠다"고 하는 것과 같습니다. 먼저 매칭 풀에 들어가는 게 우선입니다.

### 1.7 왜 기존 블로그 지수가 의미 없어졌나

기존 블로그 지수(최적블·블탭 지수)는 **블로그탭에만 영향** 을 줍니다. 새로운 AI 검색 영역(새로운 스마트블록·홈판 등)에서는 블로그 지수가 전혀 반영되지 않습니다.

- 막 만든 블로그가 AI 검색에서 1등을 하기도 함
- 오래된 최적블이 새 검색 영역에서 나락 가기도 함

AI 검색과 블로그탭은 **완전히 다른 게임** 입니다.

### 1.8 두 축으로 나눠서 봐야 한다 — "잘 쓴 글" ≠ "인정받은 채널"

새 알고리즘에서 가장 자주 오해되는 지점입니다. 흔히 "이제는 채널 권위가 중요하고 글은 덜 중요하다"라는 식으로 단순화되지만, 실제로는 **두 개의 독립 차원**이 함께 작동합니다.

| 차원 | 의미 | 본 강의 어디에 해당 |
|------|------|-------------------|
| **잘 쓴 글** (콘텐츠 품질) | 페르소나 적합도, FAQ/Q&A 구조, 제목 패턴, 키워드(주어) 사용, 작성자 권위성, 한 줄 경험 | PART 2~3 |
| **인정받은 채널** (도메인 신뢰) | 도메인 권위, 운영 이력, 공식 인증, 권위 카페·공식 블로그, 연결고리 5종 | PART 4 + PART 3.6 |

**과거 알고리즘:** 두 축에 모두 가중치 부여. **잘 쓴 글이 채널의 한계를 어느 정도 극복** 가능했음.

**새 알고리즘:** **채널 인정이 1차 필터로 작동.** 이 필터를 통과한 채널들 안에서만 글의 품질로 순위가 결정됨. 따라서:

- **글만 잘 써도 진입 안 되는 영역이 있다** (예: 학자금대출이자 — 공식 블로그가 아닌 한 권위성 자체가 없음)
- **채널만 좋고 글이 부실하면 진입은 되지만 노출이 안 된다** (목록엔 들어가도 클릭 못 받음)

즉 "전환"이 아니라 **두 축이 공존하되 채널 가중치가 크게 이동**한 상태입니다. 따라서 작업 순서도 항상:

1. **채널 진입 가능성 진단** (PART 1·2: 영역 판정 + 분포 분석)
2. **콘텐츠 품질 설계** (PART 3: 페르소나 + 글쓰기)
3. **채널 신뢰 강화** (PART 4: 외부링크·연결고리)

순으로 흘러야 합니다. 글부터 쓰기 시작하면 1단계에서 막힌 영역에 시간을 낭비하게 됩니다.

## PART 2. 분석 — 글 쓰기 전 무엇을 봐야 하나 ⭐ {#part-2}

> 💼 **마케터/대행사 관점 활용법:** 클라이언트가 "1등 만들어달라" 요구할 때, 진입 가능 영역과 불가능 영역을 데이터로 구분해서 설명할 근거가 됩니다.

### 2.1 상위 5개 분포 분석법

글을 쓰기 전, 반드시 해당 키워드를 검색해서 **강력추천영역 상위 5개(또는 6~8개)의 플랫폼 분포** 를 확인합니다.

#### 6개 키워드 실전 사례

| 키워드 | 상위 분포 | 소비자 맥락 해석 | 블로그 진입 여지 |
|--------|-----------|-----------------|-----------------|
| 탄 냄비 닦는 법 | 지식백과1 + 블로그4 | 방법을 찾는 행동형 | 있음 (HOW 중심) |
| 요즘 뜨는 주식 | 웹문서2 + 블로그3 | 전문가 분석 기대 | 있음 (전문성+경험 필요) |
| 요즘 볼만한 드라마 추천 | 웹문서2 + 블로그1 + 지식인2 | 제3자 객관 의견 선호 | 제한적 |
| 드라마 추천 | 카테고리 분류형 | "검색 기준이 모호해서 AI가 단일 의도로 묶지 못함. 여러 기준으로 분류해 다양하게 보여주는 방식으로 작동함 ('니가 뭘 기준으로 찾는지 몰라서 분류해서 다 보여줄게' 구조)" | 복잡 |
| 학자금대출이자 | 웹문서4 + 지식인1 | 공식자료·권위성 우선 | 거의 없음 (실제로 교육부 공식 블로그가 6등에 노출됨. 공식 블로그가 아닌 한 권위성 자체가 없는 키워드) |
| 눈이 침침할때 | 웹문서1 + 카페4 | 다수 의견으로 객관 판단 원함 | 개인 블로그로는 불리 |
| 강남 데이트 추천 | 지식인3 + 카페2 | 질문 + 다수 의견 동시 원함 | 개인 경험 일방 리뷰는 진입 불가 |

#### 분포 해석 공식

```
카페 3개 이상 = "다수의 의견으로 객관적 판단" 원하는 소비자
지식인 존재 = "질문에 대한 해답"을 최우선하는 소비자
웹문서 다수 = 공식 정보·권위성 중시
블로그 다수 = 개인 경험·간접 지식 선호
```

> **권위 카페란:** 인기글 영역에서 카페가 상위에 자주 등장하는 건 단순히 "다수 의견" 때문만이 아닙니다. 운영 기간이 길고 활성도 높은 **권위 카페**는 사실상의 진입 조건으로 작동합니다. 일반 블로그가 그 영역을 비집기 어려운 이유는 권위 카페 + 공식 블로그가 이미 자리를 차지하고 있기 때문입니다. 카페 多 키워드에서 "카페가 많으니 블로그로 1등 불가"를 판단하는 빠른 근거가 됩니다.

#### 실전 사례: "홈페이지제작비용"

검색 결과: **카페4 + 웹문서2**

해석:
- 카페 다수 → 다수 의견 종합 + 업체 선정 목적
- 기업 웹문서가 더 유리한 구조
- 블로그는 전부 아래로 깔림 (8·9·10등이 블로그)

결론: 이 키워드에서 블로그로 강력추천영역 진입은 사실상 불가. **6등 이하 영역을 노리는 전략** 이 현실적.

### 2.2 제목 패턴 7종

상위 글의 제목을 분석할 때 쓰는 분류 기준입니다.

| 패턴 | 의미 | 예시 |
|------|------|------|
| **HOW** | 어떻게 해야 하는가 | "탄 냄비 닦는 가장 쉬운 방법" |
| **WHAT** | 무엇을 이야기하는가 | "요즘 뜨는 주식 종목 총정리" |
| **WHO** | 누구에 관한 이야기인가 | "인물 프로필, 투자 고수는 누구" |
| **WHEN** | 시간·시점 명시 | "2025년 5월 기준 요즘 드라마 추천" |
| **THAT** | 비교·나열형 | "A vs B 비교해봤습니다" |
| **WHY** | 이유·원인 | "왜 반도체 주식이 뜨는가" |
| **DO** | 후기·실행형 | "직접 해봤더니 이렇더라고요" |

> **WHEN 패턴 — "요즘"의 상대적 시간 메커니즘:** "요즘", "최근", "이번 주" 같은 상대적 시간 단어가 들어간 키워드를 검색하는 사람은 **시간 맥락**을 가지고 있습니다. 이 키워드군에서는 제목에 **구체적인 연·월·기간**을 명시한 글이 유리합니다. "2026년 5월 요즘 드라마 추천"처럼 명시적 시간 비교군이 없으면 AI가 맥락을 읽지 못해 기존에 잘 노출되던 오래된 글을 우선 노출하는 경향이 있습니다.

> 중요: **제목의 맥락이 곧 본문의 맥락** 이어야 합니다. AI는 제목과 본문의 맥락이 일치하는지 확인합니다. 예전처럼 "서울근교가볼만한곳"이라고 제목 쓰고 서울 교통편만 언급해도 됐던 시절은 끝났습니다.

### 2.3 FAQ vs Q&A 구조 분석 ⭐

같은 키워드라도 어떤 구조로 쓰느냐에 따라 노출 가능성이 달라집니다.

| 구분 | 특징 | 언제 유리한가 |
|------|------|---------------|
| **FAQ형** | 자주 묻는 여러 질문을 목록으로 모아 각각 짧게 답변 | 키워드의 소비자가 "다양한 궁금증"을 가진 경우 |
| **Q&A형** | 하나의 질문에 집중해 깊이 파고드는 심층 답변 | 단일 질문에 명확한 답이 필요한 경우 |

#### 홈페이지제작비용 상위 1~10등 분석

| 순위 | 플랫폼 | 구조 | 특징 |
|------|--------|------|------|
| 1등 | 카페 | Q&A형 | 질문: "업체 비교 기준 알려주세요", 작성자: **중소기업 기획팀 과장** → 권위성 높음 |
| 2등 | 웹문서 | FAQ형 | "우리 회사 가격은 이렇습니다" |
| 3등 | 웹문서 | FAQ형 | "우리 회사 가격은 이렇습니다" |
| 4등 | 카페 | Q&A형 | 질문: "이 가격이 맞나요?", **작성자 권위성 없음** → 1등보다 아래 |
| 5등 | 카페 | FAQ형 | "가격 비교 기준은 이렇습니다" |
| 6등 | 카페 | FAQ형 | 다양한 문제 해결 후 최종 업체 결정 후기 |
| 8~10등 | 블로그 | Q&A형 | 비용 하나만 다루는 좁은 Q&A, AI 작성 의심 |

**블로그 분석 결론:** 블로그 간 경쟁에서도 FAQ형이 유리합니다. "비용이니까 비용만 써야지"가 아니라, "비용을 검색하는 사람이 어떤 다양한 질문을 가지고 있는가"로 범위를 넓혀야 합니다.

> **반직관 포인트 — 권위성은 답글에만 필요한 게 아니다:** "홈페이지제작비용" 1등이 기업 입장에서 기준을 알려주는 정보글이 아니라, **실무자(중소기업 기획팀 과장)가 고민을 털어놓은 Q&A형 질문 글**이었다는 점을 주목하세요. 권위성 명시는 답을 주는 글에만 필요한 게 아닙니다. 질문하는 글에서도 **작성자 직책을 밝히면** 답글을 받은 글보다 오히려 높이 올라갈 수 있습니다.

#### 상위 분포 분석 2단계 방법론

상위 5~8개를 봤다면, 분석을 두 단계로 진행하면 더 정확한 전략이 나옵니다.

**1단계 — 공통요소 추출 (카페·웹문서 포함 전체)**

상위 1~6등 전체(카페·웹문서·지식인 포함)의 공통 특성을 추출합니다. "강력 추천받는 글에는 어떤 공통점이 있는가"를 먼저 파악해야 그 영역의 소비자 의도를 정확히 읽을 수 있습니다.

**2단계 — 블로그 단독 분리 분석**

블로그만 따로 추려서 블로그 간 경쟁을 분석합니다. "어떤 구조·권위성·페르소나가 블로그에서 통하는가"가 이 단계의 핵심입니다. "홈페이지제작비용"에서 8등 블로그가 AI 작성 글임에도 뜬 이유가 FAQ 구조 때문이었다는 게 이 방식으로 발견됐습니다.

### 2.4 강력추천영역 진입 불가 시 "6등 노리기" 전략

상위 5개 중 블로그가 한 개도 없다면, 사실상 강력추천영역 진입은 불가능합니다. 이걸 바꾸려면:

- 실제 사용자 클릭 트래픽을 대규모로 끌어오거나
- 기자단·배포 블로그 수십만 개 동원해 콘텐츠 트렌드 자체를 바꾸거나
- 코카콜라급 대형 기업의 미디어 마케팅 심리전을 펼쳐야 합니다

일반 마케터에게 비현실적입니다. 이런 키워드는 **6등 이하 영역(강력추천 아래 일반 검색)을 목표** 로 전략을 세우세요.

## PART 3. 작성 — 어떻게 써야 하나 ⭐ {#part-3}

> 💼 **마케터/대행사 관점 활용법:** 클라이언트 콘텐츠를 교정·기획할 때, "왜 이렇게 써야 하는가"를 설명하는 이론적 근거로 씁니다.

### 3.1 페르소나 우선 원칙

글을 쓰기 전, **"이 글을 읽을 사람이 누구인가"** 를 먼저 정해야 합니다. 대상을 정하지 않으면 어떤 키워드로 검색할지조차 예측할 수 없습니다.

#### "서울 가볼만한 곳" 사례

이 키워드를 검색하는 사람은:
- 아이가 있는 부모일 수도 있고
- 커플일 수도 있고
- 노년 부부일 수도 있습니다

내가 "아이와 함께하는 가족여행"이라는 테마로 글을 쓰면서도 제목에 "아이와" 또는 "아이들과"를 넣지 않으면? 제목부터 이미 탈락입니다.

반대로, 페르소나를 정하면 자연스럽게 "아이와 서울 가볼만한 곳" 같은 파생 키워드에도 노출되는 보너스가 생깁니다.

#### 로봇청소기 사례

로봇청소기를 검색하는 수만 명 중 "이 사람들이 정확히 이런 제품을 원할 거야"라고 확언할 수 있을까요? 불가능합니다. 왜냐하면:

- 문턱 있는 집 vs 문턱 없는 집
- 러그가 있는 집 vs 없는 집
- 1인 가구 원룸 vs 가족이 사는 아파트

각자 상황이 다릅니다. 예를 들어 "문턱을 넘지 못하는 저가 제품"을 리뷰한다면, 타겟을 **1인 가구 원룸 자취생** 으로 좁히면 됩니다. 그 상황에서 100% 쓸 수 있는 제품이니까요.

### 3.2 페르소나 세분화 ⭐

같은 타겟 안에서도 더 뾰족하게 쪼개야 합니다.

#### 홈페이지 의뢰 사례

"홈페이지 제작하려는 기업"이라는 타겟도 사실 둘로 나뉩니다:

| 타겟 | 특징 | 글의 방향 |
|------|------|-----------|
| **결정권 없는 말단직원** | 전문 지식 부족, 결정권 없음 | "아, 그렇구나" 수준으로 끝남 |
| **결정권 있는 관리임원** | 전문 어휘에 익숙, 업체 선정 체크리스트 보유 | 어휘 수준 높이고, 체크리스트 중심으로 구성 → 영업 전환율 상승 |

결정권자를 타겟하면 어휘가 전문적으로 바뀌고, 그들의 "업체 선정 체크리스트"를 중심으로 글이 구성되면서 실제 문의·전환율이 높아집니다.

### 3.3 키워드(주어) 사용 원칙

> "키워드는 문학적으로 '주어'입니다. 내가 무엇에 대해 이야기하는가, 그게 핵심단어입니다."

키워드 반복을 줄이겠다고 **대명사("이거", "저거", "그것")** 로 치환하면 큰일 납니다.

#### 가족 대화 비유

가족끼리는 서로 맥락을 완전히 알기 때문에 "어? 어어. 어어어어."만 해도 대화가 됩니다. 그런데 AI는 처음 보는 상대입니다. 주어를 다 빼고 대명사로만 쓴 글을 AI에게 던지면?

> 길 가다 아무나 붙들고 "어? 어어어. 어어어 어. 아~ 어허~ 오호~" 이러보세요. 미친놈 취급받습니다.

AI도 마찬가지입니다. 대명사가 가득한 글은 AI가 "이거가 뭔데?"라며 맥락을 제대로 읽지 못합니다. **키워드(주어)를 자연스럽게 반복** 해서 쓰세요.

### 3.4 작성자 권위성 명시 ⭐

글에 **작성자가 누구인지 명확하게** 밝혀야 합니다. 밝히지 않으면 AI는 기본값으로 "비실무자"로 가정합니다.

"홈페이지제작비용" 키워드에서 1등이 된 카페 글의 작성자는 "중소기업 기획팀 과장"이라고 명시되어 있었습니다. 동일한 Q&A형 질문 글이라도 작성자 권위성이 명시된 글이 그렇지 않은 글보다 위에 올라간 이유입니다.

**반직관 포인트:** 권위성 명시는 답을 주는 글에만 필요한 게 아닙니다. "중소기업 기획팀 과장이 업체 비교 기준을 물어보는" **질문 글**이 1등을 차지했습니다. "이 사람이 물어볼 만큼 이 키워드가 어렵고 실무적인 고민이구나"라는 신호를 권위성 명시가 만들어냅니다. 답을 아는 전문가의 글만큼, 그 분야를 직접 다루는 실무자의 고민 글도 강력합니다.

> 글에 명확하게 누구라고 명시되어 있지 않다면 비실무자로 가정하는 게 원칙입니다.

### 3.5 단 한 줄의 경험 삽입 ⭐

블로그는 네이버 분류 기준으로 **"개인 채널"** 입니다. 카페나 지식인과 다르게, 개인의 경험이 들어가야 합니다. 단 한 줄이라도요.

"야 나도, 어 너도?"라는 **공감대 형성** 이 핵심입니다. 비슷한 글이 수백 개인 상황에서, 초입에 "저도 이거 때문에 엄청 빡쳤는데요"라고 한 줄 넣는 것만으로도 클릭률과 체류시간이 달라집니다.

법률·의료처럼 100% 정보성 글만 써야 하는 분야에서도 이 패턴이 통한다고 확인됩니다.

### 3.6 제3자 권위성 인용 ⭐

자기 주장만 내세우는 것보다 **외부 권위 있는 출처를 인용** 하면 신뢰도가 높아집니다.

| 권위성 유형 | 설명 | 예시 |
|------------|------|------|
| **자기주장 권위성** | 내가 직접 밝히는 전문성 | "저는 기획팀 과장입니다" |
| **객관적(제3자) 권위성** | 인정받은 외부 출처 인용 | 잡코리아·공공기관·기업 공식 정보 인용 |

비유: 대학 교수가 "이 책 좋다"라고 하는 것과 일반인이 "이 책 좋다"라고 하는 것은 무게가 다릅니다. 알고리즘도 같은 방식으로 작동합니다. 현실의 팩트체크 과정을 수학적 루틴으로 만들어둔 것이 알고리즘입니다.

잡코리아·기업정보 등을 인용하면 "권위 있는 사이트의 인용으로 제3자 증명"이 됩니다.

#### 도메인 권위 — 통합검색 진입의 실질 조건

통합검색에서의 진입은 사실상 **도메인 권위**(누가 운영하는 사이트인지, 네이버에 인정받았는지)에 좌우됩니다. 콘텐츠의 양이나 품질보다 **도메인 자체의 신뢰**가 먼저입니다. 아무리 글을 잘 써도 도메인 권위가 없으면 진입 자체가 막히는 구조입니다. PART 4의 "법적 근거"와 "연결고리 5종"이 바로 이 도메인 권위를 만드는 구체적 방법론입니다.

### 3.7 7단계 AI 도구 워크플로

개념 이해용 시연 기준입니다. "무조건 되는 정답"이 아니라, **원리를 이해하기 위한 참고서** 로 활용하세요.

#### 준비물

- 페르소나.txt: 페르소나 헬퍼 결과물
- 초안.txt: 내가 직접 쓴 200자 수준의 핵심 초안
- 교정안.txt → 교정안2.txt → 최종본: 단계별 개선 결과물

#### 단계별 워크플로

**1단계: 페르소나 도출**

도구: `https://gpt.datalab.tools/assistant/brand/persona`

명령어:
```
"강남 데이트 추천"을 검색하는 소비자 집단 5명의 5W1H와 Pain Point를 나열해줘.
```

결과를 `페르소나.txt`로 저장합니다.

> 부가 활용법: 페르소나 헬퍼에 검색결과 상위 글의 제목+본문 리스트를 넣어도 됩니다. "게시글의 타겟 고객 페르소나 및 5W1H + Pain Point 도출"을 요청하면 됩니다. 결과는 거의 동일하지만 시간이 더 걸립니다.

**2단계: 내 초안 직접 작성**

200자 수준의 핵심 이야기 흐름만 씁니다. AI가 생성한 글이 아니라, **내가 직접 경험한 핵심 사실** 을 짧게 적는 것입니다.

예시 (강남 데이트 추천):
```
오늘 강남에서 여자친구랑 데이트를 했는데요.
"통통통통"이라는 처음 보는 가게가 있어서 다녀왔어요.
인테리어도 아기자기하고 다른 커플도 많더라고요.
공간이 분리되어 있어서 편안하게 데이트하기 좋았어요.
가격도 합리적이고 디저트도 맛있었습니다.
```

이 초안을 `초안.txt`로 저장합니다.

**3단계: 톤앤매너 헬퍼로 1차 초벌 작성**

도구: `https://gpt.datalab.tools/assistant/brand/voice`

명령어:
```
초안의 톤앤매너를 유지하며, 페르소나의 5W1H를 충족하는, 1500자 분량의 블로그 글을 
작성자 경험 중심으로 작성. 페르소나의 Pain Point에 대한 느낀점, 공감, 꿀팁 등의 
간접경험 정보 제공을 반드시 포함해야 함.
```

결과를 저장하되, **그대로 복붙하지 말고 반드시 직접 교정** 합니다. 오류(사실과 다른 내용)를 수정한 뒤 `교정안.txt`로 저장합니다.

**4단계: 초안작성 헬퍼로 2차 교정**

도구: `https://gpt.datalab.tools/assistant/creator/write`

이 도구는 문학적 구조론과 심리 장치 기반으로 설계되어, 톤앤매너 헬퍼와 전혀 다른 기능입니다.

명령어:
```
페르소나의 5W1H와 Pain Point를 완전하게 충족할 수 있도록 교정안 내용을 지침에 따라 재교정
```

결과를 `교정안2.txt`로 저장합니다.

**5단계: 검색최적 헬퍼로 최종 최적화**

도구: `https://gpt.datalab.tools/assistant/marketing/saeo`

동일한 명령어로 진행한 후, 마지막으로 페르소나를 하나로 좁힙니다:

```
연애 중인 커플 페르소나에 집중해서 다시 작성
```

**6단계: SEO/AEO/GEO 대응 (구글 포함)**

구글 AI 검색까지 챙겨야 한다면 추가 명령어:

```
SEO/AEO/GEO에 정합하도록 재작성
```

글이 많이 바뀔 수 있습니다. 이 단계는 본인이 직접 교정하거나 추가 지시를 내려야 합니다.

**7단계: 페르소나 좁히기 (최종 확인)**

최종 글이 처음에 설정한 페르소나 한 그룹에 집중되어 있는지 확인합니다. "모든 소비자를 만족시키려는" 글은 사실상 아무도 만족시키지 못합니다.

#### 워크플로 요약표

| 단계 | 도구 | 목적 |
|------|------|------|
| 1 | 페르소나 헬퍼 | 소비자 5W1H + Pain Point 도출 |
| 2 | 직접 작성 | 핵심 초안 (200자, 사람이 씀) |
| 3 | 톤앤매너 헬퍼 | 내 말투 유지하며 1500자 초벌 생성 |
| 3-1 | 사람 교정 | 사실 오류 수정, 교정안.txt 저장 |
| 4 | 초안작성 헬퍼 | 문학적 구조·심리 장치 기반 재교정 |
| 5 | 검색최적 헬퍼 | SEO/AEO/GEO 관점 최적화 |
| 6 | 검색최적 헬퍼 | 페르소나 좁히기 최종 작성 |

## PART 4. 운영 — 외부링크와 스팸 회피 ⭐ {#part-4}

> 💼 **마케터/대행사 관점 활용법:** 클라이언트 블로그에 외부링크(홈페이지·랜딩페이지 등)를 삽입할 때, 스팸 처리·도메인 차단 없이 안전하게 운용하는 기준이 됩니다.

### 4.1 핵심 개념 — "법적 근거"

네이버는 **"법적 근거"가 없을 때 패널티를 줍니다.** 반대로, 법적 근거가 있으면 패널티를 피할 수 있습니다. 이를 **합법 울타리** 라고 부릅니다.

> "이거 다 내꺼야"를 증명하는 단계임. 모든 연결고리는 "같은 네이버 계정"으로 진행되어야 합니다.

왜 같은 계정이어야 할까요? 단순히 수집된 익명 정보로는 법리적 근거가 안 됩니다. 가입 시 넣은 실명 인증 자체가 법리적 장치가 됩니다.

### 4.2 연결고리 5종

블로그와 외부 자산을 연결하는 5가지 핵심 연결고리입니다.

| 연결고리 | 설명 |
|----------|------|
| **네이버 ID** | 모든 자산의 기본 연결점 |
| **사업자정보** | 블로그에 사업자번호 등록 + 홈페이지에도 동일하게 명시 |
| **홈페이지 도메인** | 웹마스터 도구를 통해 소유주 확인 후 등록 |
| **광고 이력** | 네이버 광고 심사 과정에서 정상 브랜드 여부 검수됨 |
| **인물정보** | 개인이라면 인물정보 등록으로 블로그와 개인정보 1:1 매칭 |

### 4.3 외부링크 안전 게재 정론 (체크리스트)

다음 순서로 진행하면 최소한의 리스크를 막을 수 있습니다.

```
[ ] 1. 네이버 블로그 사업자정보에 홈페이지 등록
[ ] 2. 네이버 웹마스터 도구에 홈페이지 등록 (소유주 확인 절차 필수)
[ ] 3. 가능하면 네이버 플레이스에도 홈페이지 등록 (사업자인 경우)
[ ] 4. 홈페이지에 "사업자번호" 명시
[ ] 5. 블로그에도 "사업자번호" 명시 (홈페이지와 동일)
[ ] 6. 통신판매업이 있으면 통신판매업신고번호도 홈페이지 + 블로그 양쪽 게재
[ ] 7. 네이버 브랜드 광고 이력 확보 (심사 과정에서 정상 브랜드 검수됨)
[ ] 8. 개인이라면 인물정보 등록으로 블로그와 개인정보 1:1 매칭
[ ] 9. 동일 네이버 계정의 인플루언서·클립·프리미엄콘텐츠는 2022년 이후 문제없음
[ ] 10. 네이버TV 등 네이버 공식 채널도 동일 계정으로 연동
```

이 과정이 곧 "이거 다 내꺼야"를 증명하는 단계입니다.

### 4.4 댓글/톡톡 우회법

#### 댓글 활용

댓글의 스팸 정책은 **블로그 글 노출 자체를 막지 않습니다.** 예를 들어, 매 글에 같은 댓글을 달아서 스팸으로 삭제되더라도 글은 정상 노출됩니다.

따라서 **댓글로 링크를 유도하는 것이 가장 안전한 방법** 입니다.

#### 네이버 톡톡 활용

블로그에 네이버 톡톡을 연동하면 스마트에디터에 톡톡 버튼이 생깁니다. 이 버튼 이미지를 원하는 대로 커스텀할 수 있습니다.

- **크기가 무제한:** 9000px짜리 이미지도 삽입 가능
- 클릭 시 네이버 공식 기능인 톡톡으로 이동 → 1:1 상담 가능
- 네이버 공식 기능이므로 스팸 문제 없음
- 챗봇 설정으로 간단한 자동화 가능
- 더 복잡한 자동화는 해피톡 같은 외부 서비스 연동 필요
- 단점: 외부 웹사이트·오픈채팅으로의 직접 유도는 불가

### 4.5 URL 구조와 스팸

외부 사이트로 링크할 때 URL 구조가 스팸 판단에 영향을 줍니다.

| URL 형식 | 스팸 위험도 | 설명 |
|----------|------------|------|
| `https://domain.com/` (메인 도메인) | **위험 없음** | 메인 페이지 랜딩은 100% 안전 |
| `/블라블라/129283` (sub path 방식) | 위험 있음 | 1가지 경로 = 1가지 역할. 반복 시 스팸 |
| `/index.php?article_id=292892` (path with params 방식) | 상대적으로 유리 | 1가지 경로가 여러 역할. 스팸에 더 자유로움 |
| 100% 동일한 링크 반복 | **스팸** | 무조건 스팸 처리 |
| 단순 리디렉션 링크 | 스팸이 되기 쉬움 | 주의 필요 |

> 참고 (정론 아님): 티스토리로 공사치는 블로그들의 실제 스팸 회피 구조는 **이중 엮음**입니다. ① 티스토리 글들끼리 내부 링크로 빽빽이 엮고, ② 티스토리에서 외부 사이트로도 엮습니다. 그리고 네이버 블로그에는 **매번 새 링크로** 올려서 100% 동일 URL 반복을 피합니다. 이 구조 자체가 스팸 감지를 우회하려는 꼼수입니다.

### 4.6 단순 판단법

복잡한 분석이 어렵다면 이 두 가지만 기억하세요:

**판단법 1:**
> 그 도메인(사이트 이름·주소)을 네이버에 검색했을 때 **대표사이트가 1등에 뜨지 않으면** 사용하지 마세요.

**판단법 2:**
> 네이버가 인정하는 사이트는 검색결과에 **"관련문서 더보기"** 가 딸려 나옵니다. "관련문서 더보기"가 없는 사이트(특히 1페이지짜리 랜딩 사이트)는 정보가치가 없는 사이트로 분류되어 스팸 처리되기 쉽습니다.

- 정보가치 없는 사이트 예시: `nocheat.datalab.tools`

### 4.7 KISA vs 네이버 약관

| 구분 | 경로 | 결과 |
|------|------|------|
| **KISA 신고** | 정부 기관 신고 | 계정 전체 사망 |
| **네이버 약관 위반** | 피싱·불법 용도 외 합법 사이트 | 도메인 단위 차단 |

피싱이나 불법 용도가 아닌 합법적 사이트라면 네이버 약관의 적용을 받습니다. 합법 울타리 안에서는 활용 가능합니다. 단, KISA 신고가 들어오면 아무리 합법 근거 기반 운영이라도 문제가 됩니다.

### 4.8 사이트 차단 시 동작

사이트가 차단될 때는 **URL 단위가 아니라 도메인(domain) 단위로 차단** 됩니다.

- 해당 사이트 이름이나 주소를 네이버에 검색해도 의도적으로 검색결과에서 배제됩니다
- 이 경우 그 링크가 삽입된 블로그 글이 노출 안 되는 현상이 발생합니다

## PART 5. 확장 — 다포 전략 + 마인드셋 ⭐ {#part-5}

> 💼 **마케터/대행사 관점 활용법:** 콘텐츠 캘린더를 짤 때, 같은 키워드로 다양한 타겟층을 공략하는 다포 전략의 근거로 씁니다.

### 5.1 다포(다중 포스팅) 문어발 전략 ⭐

페르소나를 세분화하면 **키워드 집단 자체가 갈립니다.**

| 타겟 그룹 | 검색하는 키워드 집단 |
|-----------|---------------------|
| 전문가 | 전문 용어 포함, 고급 키워드 |
| 초보자 | "~방법", "~쉬운", 기초 키워드 |
| 아무것도 몰라요 | "~뭐야", "~이게 뭔가요", 입문 키워드 |

같은 주제 하나에서 타겟별로 완전히 다른 키워드 집단이 만들어집니다. 이것이 **진짜 문어발(다포) 전략** 입니다.

기존 방식의 한계: 연관검색어·자동검색어 기준으로 검색량 기반 키워드 확장을 하면, **타겟이 계속 전문가에 고정된 채** 키워드 수만 늘어납니다. 검색량이 큰 키워드는 대부분 전문가 타겟 키워드군에 속해 있기 때문입니다. 그 결과 비전문가용 키워드에서도 전문가용 글이 작성되는 **페르소나 미스매치**가 발생합니다. 다포 전략은 페르소나별로 키워드 집단 자체가 갈리기 때문에 이 함정을 구조적으로 우회합니다.

### 5.2 글 안에서의 자기 인용

비교글·정보글 본문 안에 "이렇게 진행한 사례가 있어요"라고 언급하고 내 다른 글 링크를 넣는 것은 효과적입니다.

- 체류시간 증가 → AI 신호
- "실제 경험이 있는 사람"이라는 인상 강화
- 내 블로그 내부 네트워크 구축 효과

### 5.3 구글 vs 네이버 비교

| 항목 | 네이버 | 구글 |
|------|--------|------|
| 경쟁 범위 | 비교적 좁음 | 압도적으로 넓음 |
| 경쟁 상대 | 비슷한 시기에 만들어진 글 | 10년 전 글, 20년 전 논문, 수천만 개 커뮤니티 글 |
| 난이도 | 상대적으로 쉬웠음 (Easy 모드) | 처음부터 어려운 시장 |

네이버가 그동안 Easy 모드였던 것이 맞습니다. AI 도입으로 이제 퍼포먼스 마케팅에서 원래 쓰던 이론적 접근(타겟팅·페르소나·5W1H)이 블로그에도 필요해진 것뿐입니다.

### 5.4 정석이 답이라는 결론

> "수학적 알고리즘이라는 게 현실에서의 팩트체크 과정이나 절차를 수학적 루틴으로 만들어둔 것입니다. 결국은 '정석'이 답입니다."

정석의 원리를 이해하면 온라인·오프라인 둘 다 적용 가능한 인사이트가 생깁니다. 여기서 수학적으로 파고들면 알고리즘·데이터 분석가가 됩니다. 공식을 가져다 쓰는 것이 기업들의 알고리즘입니다.

이 시장은 **관찰 + 분석 + 고찰** 이 끝없이 필요한 시장입니다.

## 부록 A. 참고 잡담 / 인사이트 {#appendix-a}

### 마케팅 에이전시 vs 클라이언트 갈등

에이전시는 타겟팅부터 시작하는데, 클라이언트는 대체로 이렇게 옵니다:

- 특허 있으니까 자랑
- 잘 팔렸으니까 자랑
- 대표님이 대단하니까 자랑
- 캠페인 나눠서 해야 효과 큰데 한 번에 하고 싶어함

마케팅을 모를수록 자랑글을 더 선호합니다. 이런 클라이언트가 "광고 효과 못 봤다"고 불만을 가장 많이 가집니다.

블로그에서도 비슷합니다. 읽는 사람을 고려하기보다 내가 쓰고 싶은 이야기에 시야가 갇히는 경우가 많습니다.

### 데이터 분석 관점에서의 코멘트

이 내용은 본질적으로 대학 수준의 마케팅·데이터분석 교재에서 다루는 인사이트와 같습니다. 새로운 비법이라기보다는, 정통 마케팅 이론을 블로그 환경에 정제해 적용한 것입니다.

### 운영 철학

> 참고서는 있어도 정답지는 없다.

AI 검색 영역은 알고리즘과 학습 데이터가 계속 변하기 때문에 정해진 정답이 존재할 수 없는 구조입니다. 원리를 이해하면 변화에 대응할 수 있습니다.

### 키워드 변화 텀이 짧아진 이유

예전 알고리즘: 통계 쌓기 → 패턴 분석 → 성과 예측 시뮬레이션 → A/B 테스트 → 고정값 변경 (수주일~수개월 소요)

AI 알고리즘: 변경된 것만 갱신 → 전체를 재분석할 필요 없이 국소적으로 재해석 → 적용 (수 시간~하루 이틀)

### "어렵다"는 반응에 대해

이 내용이 어렵게 느껴지는 게 정상입니다. 쉽게 풀어 설명하기엔 원리를 먼저 알아야 합니다. 원리 없는 꼼수는 변칙성 플레이일 뿐이고, 네이버가 바뀔 때마다 무너집니다.

### 사이드 토픽

이모티콘 만들기 등 본 주제와 무관한 사이드 토픽이 강의 중 잠시 언급되었으나, 본문과 무관하므로 생략합니다.
